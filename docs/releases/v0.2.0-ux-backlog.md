# TaskCli v0.2.0 UX Backlog

- **日期**：2026-05-30
- **狀態**：候選 backlog
- **方向**：自用體驗優化

## 背景

v0.1.0 已完成 local-first task CLI 的核心閉環：`draft → review → finalize → track`。實際用來管理 TaskCli 自身後，下一個痛點不是資料模型，而是日常操作太依賴完整 draft/review 流程；對單一臨時任務、查找、排序、補內容、挑下一件事仍不夠快。

## v0.2.0 建議目標

讓 TaskCli 從「可建立與追蹤 task」進一步變成「每天開發時順手使用的 task CLI」。

成功標準：

1. 臨時新增一件 task 不必建立 draft。
2. task 變多後能用 query/sort/limit 快速定位。
3. CLI 能回答下一件可做的 task。
4. task 建立後能補 body / 驗收條件。
5. 安裝後能快速確認版本與從 help 得到常見用法。
