## 核心功能

### 本機 task 管理

- `taskcli init` 建立 `.taskcli/` 骨架。
- `taskcli draft create --stdin|--from-json` 建立可審閱 draft。
- `taskcli review <draft-id> --open` 啟動 localhost-only 審閱頁。
- `taskcli finalize <draft-id>` 將審閱後的 draft 轉為正式 task。
- `taskcli list/show/update/done/rm` 管理正式 task。

### Schema

正式 task 支援：

- 基本欄位：`id`、`title`、`type`、`status`、`priority`、`tags`、`created`、`updated`、`body`。
- 選填欄位：`due`、`depends_on`、`assignee`、`estimate`、`source`。

### GitHub Issues 匯入

- `taskcli import github` 透過 GitHub CLI (`gh`) 單向匯入 issue。
- 以 `source: github:owner/repo#<number>` 做冪等 upsert。
- 支援 `--repo`、`--state`、`--label`、`--limit`、`--dry-run` 與單一 issue number。

### Agent skill / 安裝

- `taskcli install-bin` 將編譯後 binary 複製到 PATH 目錄。
- `taskcli skill install` 將內建 `skills/taskcli/SKILL.md` 安裝到 Claude skills 目錄。

## 安裝與驗證

```bash
bun install
bun test
bun run build
./dist/taskcli --help
```

## 已知限制

- `import github` 需要已安裝並登入 GitHub CLI (`gh auth login`)。
- GitHub import 是單向同步；re-import 會以 issue open/closed 狀態覆寫本地對應 task 的 status。
- 目前不提供跨平台 release artifact，自行以 Bun build。
- 不自動修改 shell PATH；`install-bin` 只複製 binary 並提示確認 PATH。
